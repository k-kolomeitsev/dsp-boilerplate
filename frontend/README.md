# DSP Boilerplate — Frontend (Vite + React)

A clean frontend scaffold ready for development. Single main page with pre-configured infrastructure for API, internationalization, and SEO.

## Tech Stack

- **React 19** + **TypeScript**
- **Vite 7**
- **Tailwind CSS 4** + **DaisyUI** (UI kit)
- **React Router 7**
- **@tanstack/react-query v5**
- **Axios** + **Orval** (API hooks generation from OpenAPI)
- **i18next** (ru / en / kz / zh / ar)
- **React Helmet Async** (SEO)
- **Headless UI** + **Heroicons**

## API: Orval + Axios Mutator

Spec: `frontend/api.yaml` (OpenAPI 3.0).

Generation config: `frontend/orval.config.js`

- `client: 'react-query'` (TanStack Query v5)
- `mode: 'tags-split'`
- `mutator`: `src/helpers/custom-client.ts` (`useCustomInstance`)

HTTP client (`src/helpers/custom-client.ts`):

- Prepends `/api` prefix to all requests (without duplicating `/api/api/*`)
- Enables `withCredentials: true` for cookie-based sessions

Generated code: `src/hooks/generated/**`

> `src/hooks/generated/*` — auto-generated, do not edit manually. Make changes in `api.yaml` and run `yarn generate:hooks`.

## Internationalization (i18n)

**i18next** is configured with five languages: Russian, English, Kazakh, Chinese, Arabic.

- Configuration: `src/i18n/index.ts`
- Translations: `src/i18n/locales/*.json`
- Language detection: `?lang=` → `localStorage` → `navigator` → `htmlTag`
- RTL support for Arabic (automatic `dir` and `lang` switching on `<html>`)
- Language switcher: `src/components/LanguageSwitcher.tsx`

## SEO

Component `src/components/SEOHead.tsx` — a generic wrapper over `react-helmet-async`:

- Required props: `title`, `description`
- Optional prop: `ogImage`
- Automatic: Open Graph, Twitter Card, canonical URL, hreflang for all 5 languages
- Base URL is taken from `VITE_APP_BASE_URL` (defaults to `http://localhost:5173`)
- Forced re-render on language change

## UI Kit (DaisyUI)

**DaisyUI** is included as a Tailwind plugin for ready-made UI components.

- Integration: `src/index.css` contains `@plugin "daisyui";`
- Documentation: https://daisyui.com/components/

Additionally, `App.css` defines base styles:

- `.hp-btn` — button (variants: `--primary`, `--green`, `--outline`, `--ghost`; sizes: `--md`, `--lg`, `--xl`)
- `.hp-input` — text input
- CSS variables: `--accent-primary`, `--accent-secondary`, `--accent-tertiary`

## Local Development

### Option A (recommended): full stack via Docker Compose

Docker Compose with a reverse proxy is configured in the repository root, providing a single origin for frontend and API:

- Frontend: `http://localhost:8080/`
- Backend API: `http://localhost:8080/api/...`

### Option B: frontend development via Vite (HMR)

When running the frontend separately via `yarn dev`:

- All requests go to relative paths `/api/*`
- To work with the backend, you need a single origin (via proxy) or a configured `proxy` in Vite

## Docker

`frontend/Dockerfile` builds the frontend and serves static files via **nginx** (SPA-ready config for React Router).
